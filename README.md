# ReflectFlow

**个人经验资产管理工具** — 把每日行动沉淀成可复盘、可复用的长期成长资产。

---

│  (周/月)     │     │  (里程碑+任务) │     │  (结构化反思) │     │  (偏差分析)  │

## 快速开始

### 1. 安装

```bash
git clone <your-repo-url>
cd reflect-flow
npm install
```

### 2. 启动

```bash
npm run dev
```

打开终端输出的本地地址（通常是 `http://localhost:5173`）。

### 3. 开始使用

首次打开时，点击 **"加载示例数据"** 体验完整功能流程，或直接开始录入今日记录。

## 典型使用场景

### 场景一：设定月度目标并 AI 拆解

在 **目标** 页面创建一个目标（如"发布项目第一个版本"），填写当前状态、期望结果、可用时间。AI 会自动拆解出里程碑和每日任务，生成可执行的计划。

### 场景二：每日结构化复盘

在 **录入** 模式记录工作、学习、副业三类要点，然后进入 **复盘** 进行结构化反思：目标是什么？差距在哪里？原因是什么？下次怎么改进？

### 场景三：每周回顾偏差与趋势

在 **我的一周复盘** 查看本周目标完成情况、主要偏差原因、AI 洞察和沉淀的原则。系统会自动识别重复偏差模式并给出建议。

### 场景四：沉淀可复用的个人原则

从复盘结论中提炼原则（如"每日任务必须小于 90 分钟"），记录适用场景和边界条件，经过验证后标记为"已验证"，形成个人经验库。

## 功能概览

### 第一层：记录与复盘

| 功能 | 说明 |
|------|------|
| **每日打卡** | 记录工作、学习、副业三类要点 |
| **结构化复盘** | 五步复盘法：梳理过程 → 回顾目标 → 评估结果 → 分析原因 → 总结经验 |
| **AI 引导** | AI 生成引导问题，辅助深度反思 |
| **周/月复盘** | 自动汇总周/月数据，生成复盘报告 |

### 第二层：目标与校准

| 功能 | 说明 |
|------|------|
| **目标管理** | 创建周/月目标，关联每日记录 |
| **AI 拆解** | 自动生成里程碑、每日任务、复盘问题 |
| **每日对齐** | 每天对照目标检查完成情况 |
| **智能校准** | 偏差分析、风险预警、计划调整建议 |
| **目标冲突检测** | 识别时间、精力、方向上的目标冲突 |

### 第三层：经验资产

| 功能 | 说明 |
|------|------|
| **原则沉淀** | 从复盘结论中提取可复用原则，带适用场景和边界条件 |
| **事前沙盘** | 重要决策前的推演和风险预判 |
| **结案报告** | 目标完成后的完整复盘与经验提炼 |
| **趋势洞察** | 识别目标偏移、停滞主题、重复问题等模式 |

## 技术栈

- **前端框架**: React 19 + TypeScript
- **构建工具**: Vite
- **状态管理**: Zustand (sliced)
- **数据存储**: IndexedDB (idb) — 本地优先，无需后端
- **数据校验**: Zod
- **AI 集成**: OpenAI 兼容 API (支持流式输出)
- **测试**: Vitest + Testing Library + Playwright

### 项目结构

```
src/
├── components/          # UI 组件
│   ├── cards/           # 日记卡片、编辑器、示例数据加载
│   ├── dialogs/         # 弹窗（导出、设置）
│   ├── nav/             # 导航栏
│   ├── primitives/      # 基础 UI 组件
│   ├── stats/           # 统计图表
│   └── views/           # 页面视图
├── features/            # 业务功能模块
│   ├── coach/           # AI 复盘教练
│   ├── dashboard/       # 总览与一周复盘
│   ├── evidence/        # 证据管理
│   ├── goals/           # 目标管理
│   ├── insights/        # 趋势洞察
│   ├── preview/         # 事前沙盘
│   ├── principles/      # 原则库
│   ├── projects/        # 项目/主题管理
│   ├── reports/         # 报告生成
│   └── reviewCases/     # 复盘案例
├── lib/                 # 工具函数与类型定义
├── services/            # 外部服务（LLM、导出）
├── store/               # Zustand 状态管理（sliced）
│   └── slices/          # 各业务切片
└── test/                # 测试配置与 fixtures
```

### AI 配置（可选）

点击右上角 **设置**，配置任意 OpenAI 兼容的 LLM 提供商：

| 提供商 | Base URL |
|--------|----------|
| OpenAI | `https://api.openai.com/v1` |
| Ollama | `http://localhost:11434/v1` |
| 其他兼容 API | 自定义地址 |

不配置 AI 时，所有核心功能仍可正常使用。

### 数据导出

点击 **导出** 导出 Markdown 格式的 ZIP 包，支持自定义日期范围和目录结构。
